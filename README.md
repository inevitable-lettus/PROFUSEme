# PROFUSEme: Prostate Cancer Recurrence Prediction via FUSEd Multi-modal Embeddings

Implementation of the BCR prediction pipeline for the CHIMERA prostate cancer dataset, following the [PROFUSEme paper](https://arxiv.org/html/2509.14051v2).

> **What this repo does:** Loads pre-computed multimodal features (pathology WSI embeddings, MRI embeddings, clinical data) directly from the public CHIMERA S3 bucket, fuses them with PCA, and trains a leave-one-out logistic regression model to predict biochemical recurrence (BCR) after radical prostatectomy.

---

## What is reproduced vs. not

The PROFUSEme paper has two stages: **feature extraction** and **fusion + prediction**. This repo reproduces the second stage only.

| Stage | What the paper does | What this repo does |
|-------|--------------------|--------------------|
| **Pathology** | Extracts 224×224 and 1024×1024 patches from raw WSIs, runs a patch encoder | Loads pre-computed `.pt` feature tensors from S3 |
| **Radiology** | Registers ADC+HBV to T2W via ITK-Elastix, runs MRI encoder | Loads pre-computed `.pt` feature tensors from S3 |
| **Clinical** | Encodes 8 attributes from surgical report | Reads JSON from S3, encodes locally |
| **Fusion** | PCA per modality → concatenate → logistic regression LOO-CV | ✅ Fully reproduced |
| **Prediction** | AUC, sensitivity, specificity, ablation study | ✅ Fully reproduced |

The CHIMERA challenge dataset provides pre-computed feature files so participants don't need to re-run the expensive patch extraction or MRI registration steps. Raw WSI preprocessing code is preserved in `archive/preprocessing.py` for reference.

## Usage

### Run the notebook (recommended)
```bash
jupyter notebook PROFUSEme_Educational_Pipeline.ipynb
```

Run all cells top to bottom. The notebook:
1. Connects to CHIMERA S3 (anonymous, no credentials)
2. Loads clinical JSON, pathology `.pt`, and radiology `.pt` features for 15 patients
3. Fuses modalities with PCA
4. Runs LOO-CV logistic regression
5. Displays AUC, ROC curve, per-patient risk scores, and ablation study
6. Optionally uploads results to Google Drive (requires `client_secrets.json` + `drive_token.json`)

```bash
python3 verify_s3_paths.py
```

## Pipeline Architecture

### Data source
- S3 bucket: `s3://chimera-challenge/v2/task1` (anonymous access)
- 15 patients: `1003, 1010, 1011, 1021, 1025, 1026, 1028, 1030, 1031, 1035, 1036, 1037, 1039, 1041, 1048`

### Feature paths on S3
| Modality | S3 path | Format |
|----------|---------|--------|
| Pathology features | `v2/task1/pathology/features/` | `.pt` tensors (1024-dim) |
| Pathology coordinates | `v2/task1/pathology/features/coordinates/` | `.npy` structured arrays |
| Radiology features | `v2/task1/radiology/features/` | `.pt` tensors (320-dim), `{id}_0001_features.pt` |
| Clinical data | `v2/task1/clinical_data/` | `{id}.json` |

### Fusion & prediction
- PCA per modality (8 components each) fit only on training fold to prevent leakage
- Logistic regression (`C=0.01`, L2) with leave-one-out cross-validation
- Evaluation: AUC with 1000-sample bootstrap 95% CI, sensitivity, specificity
- Ablation study across all 7 modality combinations

### Clinical features encoded (8-dimensional vector)
| Feature | Encoding |
|---------|----------|
| Age at prostatectomy | `(age - 60) / 15` |
| ISUP grade | `grade / 5` |
| pT stage | pT2=0, pT3a=0.33, pT3b=0.67, pT4=1.0 |
| Positive lymph nodes | binary |
| Capsular penetration | binary |
| Positive surgical margins | binary |
| Seminal vesicle invasion | binary |
| Lymphovascular invasion | binary |
